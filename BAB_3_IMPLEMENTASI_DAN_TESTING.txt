IMPLEMENTASI DAN TESTING

3.1 Implementasi Hardware ESP8266

3.1.1 Konfigurasi Pin dan Komponen
Implementasi hardware menggunakan ESP8266 NodeMCU dengan konfigurasi pin sebagai berikut:
- GPIO D1-D4: Output untuk LED (Red, Green, Blue, Yellow)
- GPIO D5-D8: Input untuk Push Button (dengan INPUT_PULLUP)
- GPIO D0: Output untuk Buzzer
- Power supply: 3.3V dari USB atau external power

3.1.2 Skema Rangkaian
Setiap LED terhubung dengan resistor 220Î© ke ground, sedangkan push button menggunakan konfigurasi pull-up internal ESP8266. Buzzer terhubung langsung ke GPIO dengan ground reference.

3.1.3 WiFi Configuration dan Security
Sistem mengimplementasikan multi-WiFi support dengan prioritas:
1. UGM-Secure (WPA2-Enterprise) - Primary network
2. Personal WiFi (WPA2-PSK) - Fallback network
3. Auto-reconnection mechanism untuk network reliability

3.2 Implementasi Backend Server

3.2.1 Express.js Server Setup
Server menggunakan Express.js v4.18.2 dengan middleware configuration:
- CORS support untuk cross-origin requests
- JSON body parser untuk API endpoints
- Static file serving untuk frontend assets
- Security headers untuk protection

3.2.2 Game Management Logic
Implementasi game trigger mechanism dengan auto-reset functionality:
- 45-second timeout untuk game triggers
- Polling-based communication setiap 2 detik
- Automatic cleanup untuk expired triggers
- Error handling untuk network failures

3.2.3 Leaderboard Management
JSON-based data persistence dengan features:
- Top 100 entries limitation
- Real-time sorting berdasarkan score
- Player analytics (personal best, average, games played)
- Data validation dan integrity checks

3.3 Implementasi Frontend Interface

3.3.1 Responsive Web Design
Frontend menggunakan mobile-first approach dengan:
- CSS Grid dan Flexbox untuk layout
- Media queries untuk device adaptability
- Progressive enhancement untuk older browsers
- Accessibility features untuk inclusive design

3.3.2 Real-time Updates
Implementasi polling-based updates:
- Leaderboard refresh setiap 10 detik
- Connection status monitoring
- Game status indicators
- Error handling dan user feedback

3.4 Testing dan Quality Assurance

3.4.1 Unit Testing
Basic test suite mencakup:
- File structure validation
- JSON format verification
- Server endpoint testing
- Hardware component checks

3.4.2 Integration Testing
End-to-end testing workflow:
- ESP8266 to Server communication
- Web interface to Server integration
- Real-time data synchronization
- Error recovery mechanisms

3.4.3 Performance Testing
System performance metrics:
- Server response time < 200ms
- WiFi reconnection time < 10 seconds

3.5 Deployment dan Monitoring

3.5.1 Azure Cloud Deployment
Production deployment menggunakan:
- Azure App Service dengan Node.js 18 LTS

- Environment variable configuration
- Health monitoring dan auto-restart

3.5.2 System Monitoring
Monitoring metrics meliputi:
- Server uptime dan availability
- Request count dan response time
- Error rate dan exception tracking
- Network connectivity status

3.5.3 Maintenance dan Updates
Maintenance procedures:
- Regular backup untuk leaderboard data
- Security updates untuk dependencies
- Performance optimization reviews
- Bug fixes dan feature enhancements
