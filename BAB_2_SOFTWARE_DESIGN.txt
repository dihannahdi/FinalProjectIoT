BAB 2 - DESAIN PERANGKAT LUNAK (SOFTWARE)

Sistem Simon Says IoT mengimplementasikan arsitektur software yang terdiri dari tiga komponen utama: backend server, frontend web interface, dan firmware ESP8266. Setiap komponen dirancang dengan menggunakan bahasa pemrograman dan teknologi yang optimal untuk fungsinya masing-masing.

2.1 Algoritma Utama Game Simon Says

Algoritma Simon Says merupakan inti dari permainan yang mengimplementasikan pola memori berurutan. Algoritma ini terdiri dari:

- Sequence Generation: Menghasilkan urutan acak 1-4 (merepresentasikan warna)
- Pattern Display: Menampilkan LED sesuai urutan dengan timing 500ms ON, 200ms OFF
- Input Validation: Membandingkan input user dengan sequence yang benar
- Scoring System: Menghitung skor berdasarkan level, waktu, dan akurasi

2.2 Bahasa Pemrograman dan Platform

Backend Server: Node.js dengan Express.js Framework
- Event-driven, non-blocking I/O untuk aplikasi real-time
- NPM ecosystem untuk dependency management
- Native JSON support untuk data processing

Frontend: HTML5, CSS3, Vanilla JavaScript
- Semantic markup untuk accessibility
- Responsive design dengan Flexbox/Grid
- Modern JavaScript (ES6+) dengan Fetch API

Firmware: Arduino C++ untuk ESP8266
- Real-time performance untuk hardware control
- Rich library ecosystem untuk IoT functionality
- Direct memory management untuk optimasi

2.3 API Design dan Communication Protocol

RESTful API endpoints:
- POST /start-game: Trigger permainan dari web
- GET /check-game-trigger: Polling ESP8266 untuk game trigger
- POST /submit-score: Submit skor dari ESP8266
- GET /api/leaderboard: Ambil data leaderboard

Format JSON konsisten untuk semua responses dengan status, message, dan data fields.

Sistem ini mengimplementasikan polling-based communication untuk real-time interaction tanpa requiring persistent connections, optimal untuk environment IoT dengan network constraints.
